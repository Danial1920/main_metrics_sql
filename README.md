# 📦 Дашборд аналитики сервиса доставки

## 📌 О проекте

Данный проект представляет собой аналитический дашборд для мониторинга ключевых показателей (метрик) сервиса доставки еды/товаров. В основе лежит реляционная база данных, хранящая информацию о пользователях, курьерах, заказах и товарах.

Цель проекта — визуализировать динамику роста аудитории, оценить эффективность работы курьеров, проанализировать поведение пользователей и отслеживать ключевые финансовые и маркетинговые показатели компании.

## 🛠️ Стек технологий

* **База данных:** PostgreSQL (хранение данных, написание сложных аналитических SQL-запросов, агрегация метрик).
* **BI-система:** Redash (подключение к БД, визуализация метрик, сборка интерактивных дашбордов).

## 🗄️ Структура базы данных (Схема)

База данных спроектирована в виде «звезды»/«снежинки» и состоит из 6 основных таблиц.

### 1. Таблицы фактов (Действия)

Описывают транзакционные события, происходящие в системе.

* **`user_actions`** (Действия пользователей)
| Столбец | Тип данных | Описание |
| --- | --- | --- |
| `user_id` | INT | Уникальный идентификатор пользователя |
| `order_id` | INT | Идентификатор заказа |
| `action` | VARCHAR(50) | Тип действия: `create_order` (создание), `cancel_order` (отмена) |
| `time` | TIMESTAMP | Время совершения действия |


* **`courier_actions`** (Действия курьеров)
| Столбец | Тип данных | Описание |
| --- | --- | --- |
| `courier_id` | INT | Уникальный идентификатор курьера |
| `order_id` | INT | Идентификатор заказа |
| `action` | VARCHAR(50) | Тип действия: `accept_order` (принятие), `deliver_order` (доставка) |
| `time` | TIMESTAMP | Время совершения действия |



### 2. Таблицы измерений (Справочники)

Хранят статичную информацию о сущностях.

* **`orders`** (Заказы)
| Столбец | Тип данных | Описание |
| --- | --- | --- |
| `order_id` | INT | Идентификатор заказа |
| `creation_time` | TIMESTAMP | Время создания заказа |

* **`users`** (Пользователи)
| Столбец | Тип данных | Описание |
| --- | --- | --- |

| `birth_date` | DATE | Дата рождения |

* **`couriers`** (Курьеры)
| Столбец | Тип данных | Описание |
| --- | --- | --- |
| `courier_id` | INT | Идентификатор курьера |

* **`products`** (Товары)
| Столбец | Тип данных | Описание |

## 📊 Расчетные метрики дашборда

Для оценки эффективности сервиса были рассчитаны следующие продуктовые и финансовые метрики, представленные на графиках.

### 👥 Аудиторные метрики (Рост и активность)

* **Новые пользователи / курьеры (New Users / New Couriers)**
* **Описание:** Количество пользователей, совершивших свой первый заказ (или курьеров, принявших первый заказ) в выбранную дату. Показывает темпы привлечения новой аудитории.
* **Формула:** `COUNT(DISTINCT id)` пользователей/курьеров, для которых `MIN(time) = текущая дата`.


* **Платящие пользователи / Активные курьеры (Paying / Active)**
* **Описание:** Количество уникальных пользователей, сделавших хотя бы один заказ в этот день, и курьеров, доставивших хотя бы один заказ. Метрика ежедневной активности (DAU).
* **Формула:** `COUNT(DISTINCT id)` при условии совершения целевого действия (`create_order` / `deliver_order`) за выбранный день.


* **Общая аудитория (Total Users / Total Couriers)**
* **Описание:** Накопительный итог (кумулятивная сумма) всех пользователей и курьеров, когда-либо пользовавшихся сервисом.
* **Формула:** `Running Sum` (накопительная сумма) от новых пользователей/курьеров за весь период.



### 📦 Операционные метрики (Заказы и логистика)

* **Общее число заказов (Total Orders) и Успешные заказы**
* **Описание:** Динамика общего количества созданных заказов и доли заказов, которые были успешно доставлены (не отменены).
* **Формула:** `COUNT(order_id)` с фильтрацией по статусам `create_order` и `deliver_order`.


* **Среднее время доставки (Average Delivery Time)**
* **Описание:** Среднее время в минутах, которое проходит с момента оформления заказа пользователем до момента его доставки курьером. Ключевой показатель качества логистики сервиса.
* **Формула:** `AVG(time(deliver_order) - time(create_order))` для каждого доставленного заказа.


* **Удельное количество заказов на пользователя (Orders per User)**
* **Описание:** Показывает, сколько в среднем заказов делает один активный клиент за день. Отражает частоту использования сервиса.
* **Формула:** `Общее число заказов за день / Количество платящих пользователей за день`.


* **Удельное количество заказов на курьера (Orders per Courier)**
* **Описание:** Средняя нагрузка на одного вышедшего на смену курьера. Помогает понять, хватает ли логистических мощностей.
* **Формула:** `Общее число доставленных заказов за день / Количество активных курьеров за день`.


* **Доля пользователей с несколькими заказами**
* **Описание:** Сегментация аудитории на тех, кто сделал заказ единожды, и тех, кто возвращается. Отражает качество удержания (Retention) и лояльность.
* **Формула:** `(Кол-во юзеров с >1 заказом / Общее кол-во юзеров) * 100%`.


* **Cancel Rate (Доля отмененных заказов)**
* **Описание:** Процент заказов, которые были созданы, но в итоге отменены пользователем (или системой). Индикатор проблем в приложении или с наличием товаров.
* **Формула:** `(Количество действий 'cancel_order' / Количество действий 'create_order') * 100%`.



### 💰 Финансовые и Маркетинговые метрики (Монетизация)

* **Выручка (Revenue)**
* **Описание:** Общая сумма денег, полученная сервисом за успешно доставленные заказы по дням.
* **Формула:** `SUM(price)` всех товаров (`product_ids`) из заказов (`orders`), которые имеют статус `deliver_order` и не отменены.


* **Накопительная выручка (Running Revenue)**
* **Описание:** Кумулятивная сумма выручки за весь период наблюдений. Показывает общую динамику заработка проекта.
* **Формула:** `Running Sum` (сумма с накоплением) от ежедневной выручки.


* **Средний чек (AOV - Average Order Value)**
* **Описание:** Средняя стоимость одной корзины (заказа). Помогает оценивать покупательскую способность и эффективность кросс-сейла.
* **Формула:** `Выручка за период / Количество успешных заказов за период`.


* **ARPU (Average Revenue Per User)**
* **Описание:** Средняя выручка на одного пользователя из всей накопленной базы. Показывает, сколько денег в среднем приносит один зарегистрированный юзер.
* **Формула:** `Ежедневная выручка / Total Users` (вся база на текущий день).


* **ARPPU (Average Revenue Per Paying User)**
* **Описание:** Средняя выручка на одного *платящего* (активного) пользователя за день. Обычно ARPPU выше ARPU, так как учитывает только тех, кто реально совершал покупки.
* **Формула:** `Ежедневная выручка / Paying Users` (клиенты, сделавшие успешный заказ именно в этот день).


* **CAC (Customer Acquisition Cost)**
* **Описание:** Стоимость привлечения одного нового активного пользователя, совершившего заказ. Позволяет понять, не тратит ли бизнес на привлечение клиента больше, чем этот клиент приносит.
* **Формула:** `Суммарные маркетинговые расходы за период / Количество новых пользователей за этот же период`.


* **ROI (Return on Investment) / ROMI**
* **Описание:** Окупаемость инвестиций. Показывает рентабельность проекта или маркетинговых кампаний (сколько прибыли приносит каждый вложенный рубль).
* **Формула:** `(Полученный доход - Затраты) / Затраты * 100%`.
